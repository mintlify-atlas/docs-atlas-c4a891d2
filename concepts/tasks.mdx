## Overview

Tasks represent prepared quantum computations that are ready to execute. When you submit a kernel to a [Device](/concepts/devices), you get back a task object that encapsulates the kernel along with its arguments and execution context.

## Task Types

Bloqade Circuit provides different task types depending on the target device:

### AbstractTask

The base task structure:

```python
@dataclass
class AbstractTask(abc.ABC, Generic[Params, RetType]):
    kernel: ir.Method[Params, RetType]
    args: tuple[Any, ...]
    kwargs: dict[str, Any]
```

All tasks store:
- **kernel**: The compiled method to execute
- **args**: Positional arguments passed to the kernel
- **kwargs**: Keyword arguments passed to the kernel

### AbstractRemoteTask

For tasks that execute on remote devices:

```python
@dataclass
class AbstractRemoteTask(AbstractTask[Params, RetType]):
    """Base class for tasks generated by remote devices."""
    
    def run(self, *, shots: int = 1, timeout: float | None = None) -> list[RetType]:
        """Runs the task synchronously and returns results."""
        return self.run_async(shots=shots).result(timeout=timeout)
    
    @abc.abstractmethod
    def run_async(self, *, shots: int = 1) -> BatchFuture[RetType]:
        """Executes the kernel asynchronously and returns a Future."""
```

**Key features:**
- Execute with multiple shots
- Synchronous or asynchronous execution
- Returns a list of results (one per shot)

### AbstractSimulatorTask

For tasks that run on local simulators:

```python
@dataclass
class AbstractSimulatorTask(
    AbstractTask[Params, RetType], Generic[Params, RetType, StateType]
):
    """Base class for tasks generated by local simulators."""
    
    @abc.abstractmethod
    def run(self) -> RetType:
        """Executes the kernel and returns the result."""
    
    @property
    @abc.abstractmethod
    def state(self) -> StateType:
        """Returns the state of the simulator after running the task."""
```

**Key differences:**
- Returns a single result (not a list)
- Provides access to the simulator's internal state
- No shot-based execution model

## Batch Futures

Asynchronous remote tasks return `BatchFuture` objects:

```python
class BatchFuture(abc.ABC, Generic[RetType]):
    """Protocol for future objects that can be awaited."""
    
    @abc.abstractmethod
    def result(self, timeout: float | None) -> list[RetType]:
        """Blocks until results are available or timeout expires."""
    
    @abc.abstractmethod
    def partial_result(self) -> list[RetType | MISSING_RESULT]:
        """Returns all results available so far."""
    
    @abc.abstractmethod
    def fetch(self) -> None:
        """Fetches results that are currently available."""
    
    def done(self) -> bool:
        """Returns True if all results are available."""
    
    def cancel(self):
        """Attempts to cancel execution."""
    
    def cancelled(self) -> bool:
        """Returns True if the future was cancelled."""
```

## Usage Examples

### Synchronous Execution (Remote)

```python
from bloqade import squin

@squin.kernel
def measure_z():
    q = squin.qalloc(2)
    squin.h(q[0])
    squin.cx(q[0], q[1])
    return squin.broadcast.measure(q)

# Create a task
task = device.task(measure_z)

# Execute with 100 shots
results = task.run(shots=100, timeout=60.0)
print(f"Got {len(results)} measurement results")
```

### Asynchronous Execution (Remote)

```python
# Submit multiple tasks asynchronously
task1 = device.task(kernel1)
task2 = device.task(kernel2)

future1 = task1.run_async(shots=100)
future2 = task2.run_async(shots=100)

# Do other work while tasks execute...

# Retrieve results
results1 = future1.result(timeout=30.0)
results2 = future2.result(timeout=30.0)
```

### Partial Results

For long-running remote tasks, you can check partial results:

```python
future = task.run_async(shots=1000)

while not future.done():
    future.fetch()  # Fetch available results
    partial = future.partial_result()
    available = sum(1 for r in partial if r is not future.MISSING_RESULT)
    print(f"Progress: {available}/1000 shots complete")
    time.sleep(1.0)

all_results = future.result(timeout=None)
```

### Simulator Execution

```python
from bloqade.pyqrack import StackMemorySimulator

sim = StackMemorySimulator(min_qubits=2)
task = sim.task(measure_z)

# Run once and get result
result = task.run()

# Access simulator state
state_vector = task.state
```

## Expectation Value Tasks

Tasks that implement `DeviceTaskExpectMixin` can compute expectation values:

```python
class DeviceTaskExpectMixin(AbstractTask):
    @abc.abstractmethod
    def expect(self, observable: ir.Method[[RetType], ObsType], shots: int) -> ObsType:
        """Returns the expectation value of the observable."""
```

This is useful for variational algorithms and quantum chemistry simulations.

## Error Handling

<Warning>
  Always handle timeouts when working with remote tasks:
  
  ```python
  try:
      results = task.run(shots=100, timeout=30.0)
  except TimeoutError:
      print("Task execution timed out")
      # Handle timeout...
  ```
</Warning>

## Best Practices

<AccordionGroup>
  <Accordion title="Use async for multiple tasks">
    When submitting multiple independent tasks, use `run_async()` to execute them concurrently:
    
    ```python
    futures = [device.task(kernel).run_async(shots=100) for _ in range(10)]
    results = [f.result(timeout=60.0) for f in futures]
    ```
  </Accordion>
  
  <Accordion title="Set appropriate timeouts">
    Always set realistic timeouts based on your circuit depth and shot count. Remote devices may have queues.
  </Accordion>
  
  <Accordion title="Monitor partial results for long jobs">
    For tasks with many shots, use `partial_result()` to monitor progress and provide feedback.
  </Accordion>
</AccordionGroup>
