## What are Circuit Dialects?

Bloqade supports multiple circuit dialects, each optimized for different quantum computing use cases and hardware backends. A dialect defines the language and operations available for expressing quantum circuits.

## Choosing a Dialect

### QASM2

Use QASM2 when:
- Interfacing with existing QASM code from other quantum frameworks
- Needing a standardized circuit representation
- Working with universal gate sets (Clifford + T, rotations)
- Importing circuits from Qiskit or other QASM-compatible tools

### SQuIn

Use SQuIn when:
- Building stabilizer circuits (Clifford gates)
- Incorporating noise models and fidelity analysis
- Need efficient simulation of stabilizer states
- Working with measurements and conditional operations

### Stim

Use Stim when:
- Designing quantum error correction codes
- Working with detector and observable annotations
- Need fast stabilizer circuit simulation
- Building fault-tolerant quantum circuits

### Native

Use Native when:
- Programming neutral atom quantum computers directly
- Optimizing for hardware-native gate implementations
- Need minimal compilation overhead
- Working with analog quantum operations

### Gemini

Use Gemini when:
- Working with logical qubits in error-corrected architectures
- Need to abstract away physical qubit details
- Building high-level quantum algorithms for fault-tolerant systems
- Designing quantum error correction experiments

## Interoperability

Bloqade provides conversion passes between dialects:

```python
from bloqade import qasm2, squin, stim

# QASM2 to SQuIn conversion
qasm_kernel = qasm2.loads(qasm_string)
squin_converted = squin.passes.qasm2_to_squin(qasm_kernel)

# SQuIn to Stim conversion
stim_circuit = stim.Circuit(squin_kernel)
```

## Common Features

All dialects support:
- **Type safety**: Static type checking at compile time
- **Composability**: Build complex circuits from simple components
- **Analysis**: Extract circuit properties and statistics
- **Emission**: Generate executable code for simulators or hardware
