This guide will help you create and simulate your first quantum circuit using Bloqade Circuit. You'll learn how to use the Squin dialect to build circuits and run them on a simulator.

## Prerequisites

Make sure you have [installed Bloqade Circuit](/installation) before proceeding.

## Creating Your First Circuit

Bloqade Circuit provides multiple dialects for quantum programming. We'll start with the **Squin** dialect, which offers a Python-native interface for quantum circuits.

### Simple Bell State Example

Let's create a Bell state (maximally entangled two-qubit state) and simulate it:

<Steps>
  <Step title="Import the necessary modules">
    ```python
    from bloqade import squin
    from bloqade.pyqrack import StackMemorySimulator
    ```
  </Step>

  <Step title="Define your quantum circuit">
    Use the `@squin.kernel` decorator to define a quantum circuit:

    ```python
    @squin.kernel
    def bell_state():
        # Allocate 2 qubits
        q = squin.qalloc(2)
        
        # Apply Hadamard gate to first qubit
        squin.h(q[0])
        
        # Apply CNOT gate with q[0] as control and q[1] as target
        squin.cx(q[0], q[1])
        
        # Return the qubits
        return q
    ```

    <Note>
      The `qalloc` function allocates quantum qubits, and all qubits start in the |0⟩ state by default.
    </Note>
  </Step>

  <Step title="Create a simulator and run the circuit">
    ```python
    # Create a simulator with minimum 2 qubits
    emulator = StackMemorySimulator(min_qubits=2)
    
    # Create a task from the circuit
    task = emulator.task(bell_state)
    
    # Run the circuit
    qubits = task.run()
    
    # Get the quantum state
    rho = emulator.quantum_state(qubits)
    
    print(f"Eigenvalues: {rho.eigenvalues}")
    print(f"State vector shape: {rho.eigenvectors.shape}")
    ```
  </Step>
</Steps>

### Complete Example

<CodeGroup>
```python Bell State
from bloqade import squin
from bloqade.pyqrack import StackMemorySimulator

@squin.kernel
def bell_state():
    q = squin.qalloc(2)
    squin.h(q[0])
    squin.cx(q[0], q[1])
    return q

# Create simulator and run
emulator = StackMemorySimulator(min_qubits=2)
task = emulator.task(bell_state)
qubits = task.run()

# Extract quantum state
rho = emulator.quantum_state(qubits)
print(f"Eigenvalues: {rho.eigenvalues}")
```

```python With Measurement
from bloqade import squin
from bloqade.pyqrack import StackMemorySimulator

@squin.kernel
def coinflip():
    # Allocate one qubit
    qubit = squin.qalloc(1)[0]
    
    # Create superposition
    squin.h(qubit)
    
    # Measure the qubit
    return squin.qubit.measure(qubit)

# Run multiple times and collect statistics
emulator = StackMemorySimulator(min_qubits=1)
task = emulator.task(coinflip)
results = task.batch_run(1000)

print(f"Results: {results}")
print(f"Probability of |0⟩: {results[False]:.2%}")
print(f"Probability of |1⟩: {results[True]:.2%}")
```

```python Rotation Gates
from bloqade import squin
from bloqade.pyqrack import StackMemorySimulator

@squin.kernel
def rotation_example():
    q = squin.qalloc(3)
    
    # Apply rotation gates
    squin.rx(0.5, q[0])  # Rotate around X-axis
    squin.ry(0.5, q[1])  # Rotate around Y-axis
    squin.rz(0.5, q[2])  # Rotate around Z-axis
    
    return q

emulator = StackMemorySimulator(min_qubits=3)
task = emulator.task(rotation_example)
qubits = task.run()
rho = emulator.quantum_state(qubits)

print(f"Final state computed successfully!")
```
</CodeGroup>

## Using the QASM2 Dialect

Bloqade Circuit also supports OpenQASM 2.0 for interoperability with other quantum computing frameworks:

<CodeGroup>
```python QASM2 Basic Circuit
from bloqade import qasm2

@qasm2.main
def main():
    # Allocate quantum and classical registers
    q = qasm2.qreg(2)
    c = qasm2.creg(2)
    
    # Apply gates
    qasm2.h(q[0])
    qasm2.cx(q[0], q[1])
    
    # Measure qubits
    qasm2.measure(q[0], c[0])
    qasm2.measure(q[1], c[1])
```

```python QASM2 with Custom Gates
from bloqade import qasm2

# Define a custom gate
@qasm2.gate
def bell_gate(a: qasm2.Qubit, b: qasm2.Qubit):
    qasm2.h(a)
    qasm2.cx(a, b)

@qasm2.main
def main():
    q = qasm2.qreg(2)
    c = qasm2.creg(2)
    
    # Use the custom gate
    bell_gate(q[0], q[1])
    
    qasm2.measure(q[0], c[0])
    qasm2.measure(q[1], c[1])
```

```python Emit QASM2 Code
from bloqade import qasm2

@qasm2.main
def circuit():
    q = qasm2.qreg(3)
    qasm2.h(q[0])
    qasm2.cx(q[0], q[1])
    qasm2.cx(q[1], q[2])

# Generate QASM2 code
qasm_code = qasm2.emit.QASM2().emit_str(circuit)
print(qasm_code)
```
</CodeGroup>

## Running Simulations

Bloqade Circuit provides two main simulation backends:

### StackMemorySimulator

Best for circuits where qubit count is known in advance:

```python
from bloqade.pyqrack import StackMemorySimulator

emulator = StackMemorySimulator(min_qubits=5)
```

### DynamicMemorySimulator

Best for circuits with dynamic qubit allocation:

```python
from bloqade.pyqrack import DynamicMemorySimulator

simulator = DynamicMemorySimulator()
```

<Tip>
  For most use cases, `StackMemorySimulator` provides better performance when the qubit count is known.
</Tip>

## Available Gates

Bloqade Circuit supports a comprehensive set of quantum gates:

<CardGroup cols={2}>
  <Card title="Single-Qubit Gates" icon="1">
    - Pauli gates: `x`, `y`, `z`
    - Hadamard: `h`
    - Phase gates: `s`, `t`, `s_adj`, `t_adj`
    - Square root gates: `sqrt_x`, `sqrt_y`, `sqrt_z`
    - Rotation gates: `rx`, `ry`, `rz`
    - Universal gate: `u3`
  </Card>

  <Card title="Two-Qubit Gates" icon="2">
    - Controlled gates: `cx`, `cy`, `cz`
    - Controlled rotations: `crx`, `cry`, `crz`
    - Multi-controlled: `ccx` (Toffoli)
    - Swap gates: `swap`, `cswap`
  </Card>

  <Card title="Measurement" icon="gauge">
    - `measure`: Measure qubit in computational basis
    - `reset`: Reset qubit to |0⟩ state
    - `is_zero`, `is_one`: Check measurement outcome
  </Card>

  <Card title="Noise Channels" icon="wave-pulse">
    - `depolarize`: Single-qubit depolarizing noise
    - `depolarize2`: Two-qubit depolarizing noise
    - `qubit_loss`: Atom loss error
    - `single_qubit_pauli_channel`: Custom Pauli errors
  </Card>
</CardGroup>
